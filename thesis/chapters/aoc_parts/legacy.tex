    \paragraph{Properties of block convolution:}This operation is associative: 
    \[ A \mathbconv (B \mathbconv C) = (A \mathbconv B ) \mathbconv C \]
    However it is not commutative\red{(since composition of convolution is not)}:
    \[ A \mathbconv B \neq B \mathbconv A \]
    It is a linear application:
    \[ \lambda_1 A * x + \lambda_2 B * x = (\lambda_1 A + \lambda_2 B) * x \]

    \[ (A \mathbconv B) * x = A * ( B * x))  \]
    This can be trivially extended to convolutions with padding "valid".     \paragraph{Orthogonal matrices}

        A row orthogonal is a matrix such that $ WW^t = I $ and a column orthogonal is a matrix such that $ W^tW = I $. Trivially the transpose of a row-orthogonal matrix is a column orthogonal matrix and vice-versa.
        
        \paragraph{Row/Columns masking of a Row/columns orthogonal matrix:} If we remove rows of a row-orthogonal matrix, the resulting matrix is still a row orthogonal matrix, Same goes for a column orthogonal matrix.

        \paragraph{Composition of row/column orthogonal matrices:} The product of row (resp columns) orthogonal matrices is a row (resp. columns) orthogonal matrix. However, the product of a row ortho with a col orth is neither row nor col orthogonal.

        \paragraph{remark about convolution and row/column orthogonality:} The shape of the Toeplitz matrix of a convolution must not be confused with the shape of its kernel. Especially the Toeplitz matrix has more rows than columns if $ c_i * s^2 \geq c_o $. \red{citer existence}% A convolution is row/column orthogonal if the Toeplitz matrix that represents its linear application is row/column orthogonal.

\section{Turbocharging old methods}

    \subsection{Advantages and limitations of previous methods}

    On one side we have RKO that comes with its advantage: good handling of channel change and efficient striding, and drawbacks: it is not orthogonal when stride $\neq$ kernel size. On the other side, methods such as BCOP, SOC, ECO, LOT and Cayley handle any kernel size but channel change (when $c_i \neq c_o$) is handled with padding/channel dropping, and striding is emulated in a way that makes the convolution more costly than when striding is not applied.

    \begin{theorem}[RKO is orthogonal when stride=kernel size]
        Proof sketch: this is equivalent with applying invertible downsampling and a 1x1 conv. The 1x1 conv consists in a linear combination of the $kxkxc_i$ channels. The resulting toeplitz matrix has then a block diagonal structure. When each block is orthogonal, the whole matrix is orthogonal. The orthogonalization of each block is exactly the operation done by RKO. 
    \end{theorem}

    We make use of our ability to fuse kernels to make the drawbacks of each approach cancel out.

    \subsection{Extending parametrizations to channel change, striding and groups}

        \paragraph{Handling channels change}
            \begin{theorem}[Composition of orthogonal convolutions]\label{thm_comp}
            the composition of two row (resp column) orthogonal convolutions is a row (resp column)  orthogonal convolution. sketch of the proof: we look at the Toeplitz matrix of the orthogonal convolution, those are rotation matrices so the product of these is still a rotation matrix, ie. an orthogonal convolution.
            \end{theorem}

            
            Under some conditions, we can extend this lemma to non-square matrices. The requirement is that the underlying toeplitz matrices must have monotonic rank (ie. $A \in \mathbb{R}^{m_1\times m_2}, B \in \mathbb{R}^{m_2\times m_3}$ with $m_1 \leq m_2 \leq m_3$ or $m_3 \leq m_2 \leq m_1$. Sketch of the proof: We need to show that we can decompose any non-square orthogonal matrix as an orthogonal projection matrix and an orthogonal square matrix with a lower rank.
            Thanks to this property, following the strategy depicted in \cite{senderovich2022towards} we can then write our convolution in two parts: 
            A) when $c_i > c_o $ a 1x1 RKO convolution that reduces channels from $c_i$ to $c_o$ followed by a kxk bcop convolution with the same amount of input and output channels.
            B) when $c_i < c_o$ we start with the bcop conv and follow with the 1x1 conv.
            by doing so we can ensure that bcop parametrization is done on $min(c_i,c_o)$ which is more efficient. The bloc convolution $\mathbconv$ allows us to fuse the kernels to create a single kxk kernel.

        \paragraph{Handling strides}

        \begin{theorem}[bloc convolution and striding]\label{thm_stride} when the first convolution uses circular padding and the second convolution uses striding. We can fuse the kernel and apply striding on the resulting conv. Sketch of the proof: unfold the summation, we find the summation of convolution using the fused kernel and applied with striding. 
        \end{theorem}

        Then we can make use of this to build orthogonal conv with striding: since RKO is orthogonal when kernel size = stride, we can combine a bcop conv with kernel size (k-stride + 1) with a rko conv with kernel size = stride. The resulting fused kernel is a kxk with stride s. Again we combined 2 orthogonal convs so the resulting convolution is orthogonal, this is a composition of \ref{thm_comp} and \ref{thm_stride}.

        \paragraph{Handling transposed convolutions} 
        
        By definition, the transposed convolution implements the implicit product with the transpose of the Toeplitz matrix defined by the kernel. Hence all orthogonal methods that have 1. explicit kernel and 2. native striding, can reuse the kernel directly in the conv transpose to build an orthogonal conv transpose.

    \subsection{SOC}

        \begin{theorem}[Explicit conv exponential]

            We can rewrite the convolution exponential as follows:

            \begin{align}
                & x + \frac{K*x}{1!} + \frac{K*K*x}{2!} + \dots \\
                = & \left(Id + K + \frac{K \mathbconv K}{2!} + \frac{K \mathbconv K \mathbconv K }{3!} + \dots \right) * x            
            \end{align}
        \end{theorem}
            Then we can apply one conv instead of n convs. Note that the resulting kernel is then larger that the original one. In theory, this could unlock large speedups, but the gain is limited in practice as the implementation of convolution layers is optimized for small kernels and large images \cite{ding2022scaling}. However as we perform only a single convolution, the activations are stored only once regardless of the number of exponential iterations. In practice, this unlocks larger batch sizes.

\section{Efficient implementation of our composite approach}

    Block convolution is a special type of convolution where the Hadamard product is replaced by a matrix product. To the best of our knowledge the only differentiable implementation is available in the implementation of BCOP \cite{li_preventing_2019}. However, since there is no cuda kernel available for block convolutions the authors relied on nested loops to construct the convolution and used the pytorch's matmul to perform matrix multiplication on GPU. Unfortunately this approach does not leverage the fact that the nested loops could be paralleled. Also, pytorch does not allow easily to parallelize cuda operators (that are already paralleled).

    \paragraph{leveraging 2D convolution to compute block convolution:}
    \red{Given two convolution kernels $A\in{K}_{c_o,c,l_1,l_2}$ and $B\in{K}_{c,c_i,k_1,k_2}$, the block convolution $R=A\mathbconv B\in{K}_{c_o,c_i,(k_1+l_1-1),(k_2+l_2-1)}$ can be implemented as proposed in \cite{bcop} by unrolling all the loops and explicit the full summation:}
    In order to efficiently implement this operation we can unroll all the loops and explicit the full summation:
    \[
        R_{i,j,x,y} = \sum_{i' \in 0}^{k-l-1}\sum_{j' \in 0}^{k-l-1}\sum_{x' \in 0}^{n}\sum_{y' \in 0}^{n} A_{i,j',x,y'} * B_{i-i',j-j',x',y}
    \]
    \red{
    \[
        R_{m,n,i,j} = \sum_{k \in 0}^{c-1}\sum_{i' \in 0}^{k-l-1}\sum_{j' \in 0}^{k-l-1} A_{m,k,i',j'} . B_{k,n,i-i',j-j'}
    \]
    }
    % By transposing dimensions in $A$ and $B$, we can turn this summation into
    \red{Inspired by \cite{wang_orthogonal_2020}, which aimed to compute $A\mathbconv A^T$ to proove orthogonality. We propose to replace the computation $A\mathbconv B$  by a convolution with zero padding between $A$ and $B^T$.}
    By re-ordering the summation, this operation can be re-written to use a 2D convolution at its core. While the formal definition of convolution only allows to perform the convolution. Transpose convolutions allow to properly handle the "full" padding being required. The depth and channel dimension can then be used to perform the matrix multiplication, which is thanks to a transposition in the second matrix such that the element-wise product corresponds to the matrix product (padding "valid" is used to ensure that no convolution is performed on the "depth" axis, since both the input and the kernel have the same length on this axis. The hole process is described in the fig\ref{fig:code}.

def fast_matrix_conv(m1, m2, groups=1):
    # m1 is m*n*k1*k2
    # m2 is nb*m*l1*l2
    m, n, k1, k2 = m1.shape
    nb, mb, l1, l2 = m2.shape
    assert m == mb * groups
    # Rearrange m1 for conv
    m1 = m1.transpose(0, 1)  # n*m*k1*k2
    # Rearrange m2 for conv
    m2 = m2.flip(-2, -1)
    # Run conv, output shape nb*n*(k+l-1)*(k+l-1)
    r2 = torch.nn.functional.conv2d(m1, m2, 
           groups=groups, 
           padding=(l1 - 1, l2 - 1))
    # Rearrange result
    return r2.transpose(0, 1)  # n*nb*(k+l-1)*(k+l-1)
    \end{verbatim}

    \subsection{Further parallelism}

        Although the bloc convolution is at the core of BCOP, it is not the only costly operation involved: the orthogonalization procedure is also very costly. In particular the algorithm require the orthogonalisation of $ 2k $ matrices of shape $ c_m \times  \frac{c_m}{2} $ plus a matrix $ c_i \times c_o $. The successive application of two orthogonalization is much more expensive than the orthogonalization of $ 2k + 1 $ matrices. As the shape of the matrices mismatch, one can wonder if the construction of a $ c_m \times c_m $ is possible using multiple $ c_m \times  \frac{c_m}{2} $ matrices. To do this, we extended the construction of orthogonal matrices by multiplying householder matrices. We follow the original approch but instead of constructing $c_m$ Householder matrices using $c_m$ normalized vectors, we use only 2 $ c_m \times \frac{c_m}{2} $ matrices: $ A_t = A_{t-1} ( I - 2 MM^t ) $ where M is an partially orthogonal matrix. 

        In the original paper, the authors depicted an iterative algorithm to construct the orthogonal kernel. However one can notice that since the bloc convolution operation is associative we can rearrange all products in order to represent the algorithm as an associative parallel scan. Then by running operations in parallel, we can reduce the time complexity from $ k $ to $ \log k $ steps.
