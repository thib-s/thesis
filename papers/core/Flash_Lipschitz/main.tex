Orthogonal convolutional layers are valuable components in multiple areas of machine learning, such as adversarial robustness, normalizing flows, GANs, and Lipschitz-constrained models. Their ability to preserve norms and ensure stable gradient propagation makes them valuable for a large range of problems. Despite their promise, the deployment of orthogonal convolution in large-scale applications is a significant challenge due to computational overhead and limited support for modern features like strides, dilations, group convolutions, and transposed convolutions.
In this paper, we introduce \textbf{AOC} (Adaptive Orthogonal Convolution), a scalable method that extends the work of \cite{li_preventing_2019}, effectively overcoming existing limitations in the construction of orthogonal convolutions. This advancement unlocks the construction of architectures that were previously considered impractical. We demonstrate through our experiments that our method produces expressive models that become increasingly efficient as they scale.
To foster further advancement, we provide an open-source python package implementing this method, called \fulllibname.%available~\libname.% ~and installable via \texttt{pip install orthogonium}.
%To accelerate innovation in this area, we have released an open-source library implementing our approach, available \libname.


    % Orthogonal layers in neural networks are known to improve robustness, limit the exploding and vanishing gradient problem, and have applications in normalizing flows, Wasserstein distance estimation, and RNN stabilization. However the construction of orthogonal convolutions is a still a challenge: exisitng methods suffers limitations that prevent their efficient application on large scale problems. We propose a fast and scalable method to build orthogonal convolution layers that support modern convolution options, such as stride, transposition, groups, and dilation (and all compositions). This unlock the construction of modern CNN architecture like Unets. We finally shows that this implementation is expressive and become more efficient with scale.
    % The block convolution between two tensors $A \mathbconv B$ is a convolution using matrix multiplication rather than scalar multiplication. Amongst other properties, block convolution enjoys the fact that $A * ( B * x )) = (A \mathbconv B ) * x$. In other words, a composition of convolutions can be fused by applying the block convolutions between the two kernels. However, the practical implementation in deep learning frameworks remains under-explored, resulting in inefficient implementation that does not make full use of GPU/TPU architectures. This paper proposes an efficient and scalable method to compute this operation. We apply this to improve existing methods that construct orthogonal convolutions, namely BCOP \cite{li_preventing_2019} that explicitly use block convolutions, we also show that skew orthogonal convolutions \cite{singla_skew_2021} can be rewritten using this framework, unlocking their applications at scale.
